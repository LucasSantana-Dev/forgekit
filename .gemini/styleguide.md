# Gemini Code Assist Style Guide

- Treat this repository as a toolkit for AI-assisted development, not as an application product.
- Keep guidance portable across tools whenever possible.
- Prefer small, reviewable changes over broad rewrites.
- Preserve the separation between:
  - rules
  - patterns
  - portable skills
  - tool implementations
  - agent organizations
- When editing documentation, optimize for beginners without removing important operational knowledge.
- Prefer objective wording over marketing language.
- Verify repository changes with the real validation commands before declaring success.
- Keep security guidance explicit and conservative.
