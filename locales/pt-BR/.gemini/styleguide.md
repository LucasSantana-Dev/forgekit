# Guia de Estilo do Gemini Code Assist

- Trate este repositório como um toolkit para desenvolvimento assistido por IA, não como um produto de aplicação.
- Mantenha as orientações portáveis entre ferramentas sempre que possível.
- Prefira mudanças pequenas e revisáveis a reescritas amplas.
- Preserve a separação entre:
  - regras
  - padrões
  - skills portáveis
  - implementações de ferramentas
  - organizações de agentes
- Ao editar a documentação, otimize para iniciantes sem remover conhecimento operacional importante.
- Prefira linguagem objetiva em vez de linguagem de marketing.
- Verifique as mudanças do repositório com os comandos reais de validação antes de declarar sucesso.
- Mantenha a orientação de segurança explícita e conservadora.
